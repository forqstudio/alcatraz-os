Add a WSL 2 NixOS configuration that produces an importable tarball via `nixos-wsl`. Extract shared settings into a common module to keep both configs DRY.

### Final directory structure

```
src/
├── flake.nix                          # Updated: add nixos-wsl input + wsl config output
├── flake.lock
├── modules/
│   └── common.nix                     # NEW: shared settings (locale, timezone, users, nix)
├── alcatraz/
│   ├── configuration.nix              # Updated: import common.nix, keep only hardware-specific stuff
│   └── hardware-configuration.nix
├── wsl/
│   └── configuration.nix              # NEW: WSL-specific config, imports common.nix
└── home/
    ├── home-common.nix
    ├── home-dev.nix
    └── home-al.nix
```

### Step-by-step changes

#### 1. Create `src/modules/common.nix` (new file)

Extract from the current `alcatraz/configuration.nix`:
- Timezone (`Europe/Amsterdam`)
- Locale settings (en_US.UTF-8 + nl_NL.UTF-8 extras)
- User definitions (`dev` and `al`, both in `wheel` group)
- `nixpkgs.config.allowUnfree = true`
- `nix.settings.experimental-features`
- `system.stateVersion = "25.11"`

**Not** included (hardware/desktop-specific):
- Bootloader, hardware imports, NetworkManager
- X11/XFCE/LightDM, PipeWire, CUPS, Firefox

#### 2. Create `src/wsl/configuration.nix` (new file)

```nix
{ config, pkgs, inputs, ... }:

{
  imports = [
    ../modules/common.nix
  ];

  wsl.enable = true;
  wsl.defaultUser = "dev";

  networking.hostName = "alcatraz-wsl";
}
```

Key points:
- Imports `nixos-wsl` module (via flake) and our `common.nix`
- No bootloader, no hardware-configuration, no display manager
- `wsl.enable = true` sets up the WSL kernel, init, interop, and binfmt
- `wsl.defaultUser` sets which user WSL logs in as
- Hostname differentiated as `alcatraz-wsl`

#### 3. Update `src/alcatraz/configuration.nix`

- Add `import ../modules/common.nix` to the imports list
- Remove the settings now in `common.nix` (timezone, locale, users, allowUnfree, experimental-features, stateVersion)
- Keep everything hardware/desktop-specific (bootloader, GRUB, X11, XFCE, PipeWire, CUPS, Firefox, NetworkManager)

#### 4. Update `src/flake.nix`

- **Add input**: `nixos-wsl.url = "github:nix-community/NixOS-WSL";` with `nixos-wsl.inputs.nixpkgs.follows = "nixpkgs";`
- **Update outputs function** signature to include `nixos-wsl`
- **Add** `nixosConfigurations.wsl` alongside the existing `alcatraz`:

```nix
nixosConfigurations.wsl = nixpkgs.lib.nixosSystem {
  system = "x86_64-linux";
  specialArgs = { inherit inputs; };
  modules = [
    nixos-wsl.nixosModules.default
    ./wsl/configuration.nix
    home-manager.nixosModules.home-manager
    {
      home-manager.useGlobalPkgs = true;
      home-manager.useUserPackages = true;
      home-manager.users.dev = import ./home/home-dev.nix;
      home-manager.users.al = import ./home/home-al.nix;
    }
  ];
};
```

### Build & usage

```bash
# Build the WSL tarball
nix build ./src#nixosConfigurations.wsl.config.system.build.tarballBuilder

# Run the tarball builder
./result/bin/nixos-wsl-tarball-builder

# On Windows, import into WSL
wsl --import AlcatrazOS C:\AlcatrazOS nixos-wsl.tar.gz
wsl -d AlcatrazOS
```

### Notes

- The `networkmanager` group in user `extraGroups` will be dropped in `common.nix` since WSL doesn't use NetworkManager. The `alcatraz` config can add it back per-user.
- `vscodium` in `home-dev.nix` will work under WSLg if a display is available, no changes needed.
- The `nixos-wsl` module handles: custom WSL kernel, `/init` setup, Windows interop (running `.exe`), `/mnt/c` automount, and native systemd support.
